# mycelium-lite — Pre-Mortem（v2 轉型）

> date: 2026-05-31
> 針對兩個**高不可逆決策**做失敗倒推：① 取代舊「連結浮現」定位 ② 退成純前端（棄 Tauri/Rust、封存本地 embedding）。
> 方法：假設 **3 個月後（2026-08-31）打開這專案，發現它失敗/被放棄了**，倒推死因，按「殺傷力 × 機率」排。

## 最高風險假設（riskiest assumption）

整個轉型的價值壓在一個**沒驗證的推論**上：

> **「概念分布不均 = brainstorm 盲點」。**
> spike 只證了「觀察到分布有差」（樣本1 c4 偏薄），**沒證**「人會因此補到真盲點」——薄概念可能只是本來就次要。

**待測問題句**：看著概念分布圖，我會不會真的補進一個原本沒想到的方向？
**判定**：自己跑 3 次真實 sprint，連 3 次都不會 → 轉型核心價值不成立，及早回頭。

### 驗證結果（2026-05-31，N0 第 1 次）

N0 跑 v2a 樣本，使用者判定「**薄得合理**」→ **訊號①（概念分布 = 盲點）否證**。
- 範圍：只否證訊號①，**不否證整個定位**；「逐字稿→積木牆」結構整理價值仍在。
- 機制說服力：同源於 spike 否證「空透鏡」——AI 均勻填充不製造突兀訊號，故無需補滿 3 次。
- 轉向：賭注移到 E2 v3「**維度覆蓋缺口**」（人手動發散時自己沒察覺的漏 = unknown unknown），待 N2 驗。詳見 `Design.md §4 E2 演進史`。

| # | 死因 | 早期警訊 | 預防/緩解 | 可逆性 |

| **A4** ⭐ | **盲點偵測是偽價值**：分布圖看了沒啟發，「薄概念=想得少」推論不成立 | 自己用幾次，沒有「啊我確實漏了」的時刻 | MVP 先做最簡分布呈現，自己跑 3 次真實 sprint 驗證；無洞見就退回「純結構視覺化」誠實承認 | 高 |

## 對 MVP 範圍的回饋（最重要）

最高風險假設 A4 **便宜可驗**，SOP 要求「花 20 小時做原型前先確認核心假設」。所以里程碑排序：

> **最小可驗證單元 = 貼一份真實逐字稿的 JSON → 渲染積木 + 概念分布 → 自己判斷「有沒有暴露到我沒想到的洞」。**
> 這一步做出來、自己跑 3 次，就能否證/確認整個轉型。其餘（人工發散、火花、收斂、匯出）都排在這之後。

見 `Design.md §8 里程碑`（N1 = A4 驗證點，是轉型生死關）。
