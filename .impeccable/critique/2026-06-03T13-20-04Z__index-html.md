## Anti-Patterns Verdict

不像 AI 做的。有明確觀點(菌絲 teal/amber、Fraunces 襯線標題、密集桌面工具、白話詞彙、有目的動效)。產品 slop 測試大致通過。靠近「dark-mode AI 工具」第二序套路,但菌絲漸層 + 襯線 + 原創互動模型把它推離通用感。低 slop。

Deterministic scan(2 warning):overused-font Fraunces = 半成立(僅 display 用,合理);single-font = 誤報(實際三族字體,偵測器只看 head 第一個 family)。
Visual overlays:環境無瀏覽器自動化,無 user-visible 疊圖;以原始碼靜態審查 + CSS 對比手算替代。

## What's Working

1. 領域誠實的詞彙 + 具體角度問句:白話原則貫徹,無技術詞外洩;角度 hint 直接降 intrinsic load。
2. 有 token、有 kill switch 的動效:--ease/--dur 收斂、揭露 stagger、gapPulse 引導、全域 reduced-motion 連 delay 歸零並註明避免卡隱藏態。
3. 互動模型本身:藏 AI → 逼手動 → 揭露差集,UI 結構(accordion + rail + reveal)真支撐概念。

## Priority Issues

[P1] 主要按鈕對比未過 WCAG AA（~3.1:1）
- accent 字 #5dcaa5 on accent-dim 底 #0f6e56,字級 12-13px/500,對比約 3.1:1,AA 要 4.5:1。.solid-btn/#parse-btn/.reveal-btn/.graph-update/.spark-take/.step-tab.active 全踩線。
- Fix:字改近白(--text on accent-dim >7:1)或壓深 accent-dim 或字 ≥14px bold。
- Command:$impeccable colorize

[P1] 鍵盤無法操作收斂步驟的勾選
- quadrant.ts .quad-item input[type=checkbox]{display:none} 會把原生 checkbox 移出 tab order/無障礙樹,鍵盤建不了 shortlist;落點純 pointer 拖曳無鍵盤替代。WCAG 2.1.1 不過,且與程式碼註解矛盾。
- Fix:改 appearance:none 或 opacity:0;position:absolute 保留焦點;落點補方向鍵微調。
- Command:$impeccable harden（或先 audit）

[P1] 首次開啟空白,工具不自我介紹
- 無存檔時 #output 全空,新使用者看不到流程與產出。README 有講但 app 沒有。
- Fix:#output 放首次 empty state(三步縮圖 + 「載入範例」鈕直接 parse 內建 demo JSON)。
- Command:$impeccable onboard

[P2] 巢狀卡片堆疊
- 想法牆 .wall-col(卡)裝 .brick(卡);步2 .work-sidebar > .dv-section > .dv-lensblock 三層卡。nested cards are always wrong。
- Fix:wall-col 退成欄語意(去 bg/border 留欄距+標頭底線);dv-section 外框改分隔線。
- Command:$impeccable layout 或 distill

[P2] body{zoom:1.25} 把單一顯示偏好寫死進產品
- 硬鎖 125%,zoom 非標準(Firefox 近期才支援),對其他 DPI 過大,推理失真。
- Fix:改 root font-size + rem 級距,或把縮放留在瀏覽器層級。
- Command:$impeccable adapt

## Persona Red Flags

Sam(無障礙/鍵盤):收斂步驟勾選框不在 tab order,建不了候選;chip 只能滑鼠拖;主按鈕字 3.1:1 偏淡。
Alex(效率派):只有 Ctrl+Enter,無全選候選、無快捷切步、2×2 不能鍵盤微調。
Mei(專案專屬,sprint 中單人使用者):從①到③要手動搬兩次 JSON、中間無進度感;ChatGPT JSON 略歪時不確定整理成功沒,只靠狀態列小字。

## Minor Observations

- .brick-quote border-left:2px 命中 side-stripe 形但是引文慣例、中性色,邊界可接受,別擴散。
- 玻璃抽屜 .work-sidebar/.dv-rail backdrop blur+44px 陰影:因是真覆蓋浮層有 elevation 理由可留,但別再加。
- .wall-col:hover 1px border+18px 陰影 = ghost-card 形(暫態),陰影可收到 ≤8px。
- 10-11px 小字(field-hint/spark-label/col-flag),×zoom 後 ~12.5-13.75px 仍偏小,核對對比。
- composer textarea placeholder 未顯式設色,建議比照 .quad-filter::placeholder 明設 text-dim 確保 4.5:1。

## Questions to Consider

- 第②步「離開去 ChatGPT 再回來」的斷點能否在 app 內給時間軸,讓人不覺得掉出流程?
- 2×2 落點的精確 x/y 真的重要,還是只需「落在哪一格」?若後者,拖曳可降級成點四格,順手解掉鍵盤可達性。
- 首次體驗一顆「載入範例」是否比任何說明文字都快讓人懂這工具?
